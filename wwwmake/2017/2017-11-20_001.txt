Hello,
    You're more than welcome.

    About hoe it's a dev / build only use

$ gem install hoe

   And than you can list all task in the project eg.

$ rake --tasks    # or $ rake -T

    Little article about hoe (rake tasks) here ->
http://planetruby.github.io/gems/hoe


> And there are no .gemspec files. How would I build / rebuild the pluto gems?

    If you want to build a local test version try / use:

    $ rake gem

   If you want to install too

    $ rake install_gem


> And do I need to do that as a step in seeing the results of code changes on a pluto site?
> or what's a sensible development workflow to do that?


   Good question. Depends on what you try to do.

    If you change the templates you do NOT need to change the ruby
code and push / publish a new gem update.

   If you're interested in adding more functionality than you will
need to push / publish a new gem update.

  You can, however, of course, first test your changes locally or
using your own"custom" gem version from github
   (instead of the "official" rubygems version).

   You're more than welcome. and I'm happy to assit. Cheers.
