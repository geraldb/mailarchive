> As far as I can tell, it's just CSS magic right?
> If I define the right classes appropriately, then it should work in
> s6, right?

   It's a little more involved. You will need to add some JavaScript
magic for hiding and showing parts in pages and change the keyboard
handler (e.g. page up, page down to get to the next slide but space or
right arrow might just be next step) and so on.

  If you want to get started it might be best to grab the code from
original S5 script [1] and move it to jQuery. Most of the action
happens in the function go().

  Also note that the FullerScreen slide show machinery supports steps
as an alternative to S5. Learn more about FullerScreen @
http://slideshow.rubyforge.org/#fullerscreen

   Cheers.

[1] http://slideshow.rubyforge.org/svn/trunk/lib/templates/s5/slides.js
