   I updated the slideshow gem (- that is, version 0.7.3 -) to remove
th dependency on Maruku (and Oniguruma) and allow and check for
optional installed Markdown libraries/gems.

  For now the search order for markdown libraries is first come, first
serve, that is: rdiscount > rpeg-markdown > maruku > bluecloth
(fallback, always present because it's required in gem spec).

  If you want to use rdiscount as your markdown library install it
using RubyGems e.g.

  gem install rdiscount

  Same goes for rpeg-markdown or maruku.

  In an upcoming release/update I plan to add a command line switch or
config settings or similar that lets you finetune and override the
markdown library selection. Not yet settled on what's the best
practice.

  Cheers.

PS: If you want to check/troubleshoot what markdown library gets
picked up by slideshow add the -v/--verbose switch and look for:

DEBUG: Markdown library rdiscount not found. Use gem install rdiscount
to install.
DEBUG: Markdown library rpeg-markdown not found. Use gem install
rpeg-markdown to install.
DEBUG: Installed Markdown libraries: maruku, bluecloth
DEBUG: Using Markdown library maruku.
