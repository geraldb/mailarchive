Hello Bruce,

   I uploaded a new slideshow gem (version 0.7) to RubyForge that now
supports custom template packages using manifests (simple text files
listing all files). It might take a couple of hours until the gem is
available according to the RubyForge service.

   What's new? How does it work?

   As usual use the -g/--generate switch to generate a sample template
package w/ manifest to get started (now using, of course, the new
packaging format). Example:

   slideshow -g        # generates a template package w/ manifest in
your working folder (using the s6 package)
   slideshow -g -s5  # same as above but uses builtin s5 template
package

   Note, you can, of course, also create your own template packages w/
manifest from scratch.

  You can find the manifest samples online:

   s5.txt.sample   @ http://slideshow.rubyforge.org/svn/trunk/lib/templates/s5.txt.sample
   s6.txt.sample   @ http://slideshow.rubyforge.org/svn/trunk/lib/templates/s6.txt.sample


   To use your own custom package use the new -t/--template switch
passing along the manifest. Example:

   slideshow -t  s6.txt  microformats
   slideshow -t  s6nosvg.txt   microformats

   Using the new package format you're invited to create your own
template package w/ manifest that uses your own extended/changed
JavaScript code and drops the svg background gradient etc.

   Let me know if you try it and have any questions or comments.
Cheers.
