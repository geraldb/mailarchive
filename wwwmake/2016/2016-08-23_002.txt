Thanks for reply, I'm really set on the tight jekyll integration so that I 
can deploy my site easily.

I finally ended up following this 
option http://luugiathuy.com/slides/jekyll-create-slides-with-revealjs/ - 
which uses reveal.js for the presentation
Which was still not exactly right because it uses client side rendering of 
markdown (which failed to parse some complicated code examples)
So I ended up with:

 
> <section markdown='1'>
> ## Slide title
> - foo
> - bar
> </section>
>
 

> <section markdown='1'>
> ## Another slide
> - bar
> - foo
> </section>


It is not as nice as markdown as with slideshow, but still acceptable for 
me, and works with with jekyll out of the box.

Regards, 
 Odin


On Tuesday, August 23, 2016 at 9:50:04 AM UTC+2, Gerald Bauer wrote:
>
> Hello,
>     If you want to add your slideshows to your jekyll site just add the
> output from 
>
>   $ slideshow build talk.txt -o o
>
>   to your jekyll site (e.g. all the html, css, js, and images) That's it. 
> (Sorry there's no "magic" all-in-one integration with jekyll serve or 
> something.) See an example here.[1] If you have more than one slideshow and 
> you're using the same slideshow package (e.g. s6, deck.js, etc.)  you can 
> (re)use the shared parts.
>
>  Cheers.
>
> [1]  https://github.com/slideshow-s9/demos
>
> Note: Not a "full" Jekyll-site but hosted on GitHub Pages. Just add your 
> talks as a (sub)folder e.g. talks/ (instead of demos/) to your Jekyll site. 
>
> PS: Using Jekyll to build the slideshow output itself is for theme 
> developers e.g. you can test / design your theme with Jekyll, that is, it's 
> just a Jekyll theme. Hope that clears up the confusion.
>
