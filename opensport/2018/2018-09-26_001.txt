Hello,
   Welcome to the football.db.

   You can read the world cup datasets into your (SQL) database of choice 
e.g. postgresql. However, you, will have to learn to use a (build) script 
on the command line (in ruby). Example:

 build.rb:

require 'sportdb/models'
require 'datafile'


DB_CONFIG = {
  adapter:  'postgresql',
  encoding: 'unicode',
  username: 'you',
  password:  'topsecrect',
  pool:      5,
  timeout:   5000
}


ActiveRecord::Base.establish_connection( DB_CONFIG )


SportDb.create
SportDb.read_builtin


datafile = Datafile::Datafile.load_file( "./worldcup.rb" )

datafile.dump

datafile.download    ## download zips from github - saved to ./tmp
datafile.read        ## read into / import into sql database


   If you run this build script with ruby and you install the ruby 
libraries / gems (sportdb-models, datafile) and if you use the worldcup 
datafile, you're all set.

  $ ruby ./build.rb

   Sorry, for now that looks like the "easiest" way. Good luck. Cheers. 
Prost.


PS: Might be easier to use SQLite than you can use all the pre-configured 
command line tools and run:

$ sportdb new worldcup    # to get an up-to-date build sport.db SQLite 
database
