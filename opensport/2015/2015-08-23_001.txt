Hello,
    As a follow-up to get you started with the import into an SQL database:

   The easiest way is to use a Datafile [1], for example, create a new
directory/folder
   and put in an empty Datafile.

   Now add something like:

  inline do
    WorldDb::Model::Country.parse( 'ru', 'Russia', 'RUS', '1 km=C2=B2', '1'=
 )
  end

  football 'Kaspi/ru-russia'

'
  Now try to run it.

  $ sportdb --verbose build            # will default to use the
(local) Datafile; use the verbose debug switch

  It should download your datasets as a zip archive.

  As a next step you will have to add a setups/all.txt to your repo e.g.

setups/all.txt:

   ru-russia!/leagues
   ru-russia!/teams
   ru-russia!/2014-15/rflp
   ru-russia!/2015-16/rflp

 And than retry (best to clean out the local directory e.g. remove
sport.db etc.) and the import should start working for real - next up
is fixingthe remaing issues poping up. For example, rename your
russian flp from teams.yml  to rflp.yml in 2014-15 and 2015-16 and
make sure you include a fixtures block (matching your fixtures/match
schedule/results files) e.g.

fixtures:
-  rflp-i
-  rflp-ii

  Let us know how it goes or if you continue. Keep it up. Cheers.

[1] github.com/openfootball/datafile
