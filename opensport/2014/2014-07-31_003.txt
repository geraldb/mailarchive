Hello,

   Hopefully you're not shocked. I reworked the code in the addon
(that is, football.db.stats).  It's now a "real" gem, that is,
footballdb [1].  Still needs a little more polish.

   What's new?

I changed the format from an all-in-one squads file e.g.:

(25) Sean Johnson
$$ pos: G, starts: 19, subIns: 0, saves: 53, goalsConceded: 33,
foulsCommitted: 0, foulsSuffered: 2, yellowCards: 0, redCards: 0,
wins: 0, losses: 0, draws: 0,

(16) Greg Cochrane
$$ pos: D, starts: 11, subIns: 0, totalGoals: 0, totalShots: 4,
shotsOnTarget: 1, goalAssists: 2, foulsCommitted: 6, foulsSuffered: 2,
yellowCards: 2, redCards: 0,


  As suggested using two files (formats):

The squads work as is, example, e.g. chicago.txt [2]:

(25)  GK   Sean Johnson
(16)  DF   Greg Cochrane
(24)  FW   Quincy Amarikwa

  And an all new format using just CSV (comma-separated values) for
the stats, example, e.g. chichago.csv [3]

Name, Pos, Starts +SubIns, Saves [GK], GoalsConceded [GK], TotalGoals,
TotalShots (ShotsOnTarget), GoalAssists, FoulsCommitted-FoulsSuffered,
YellowCards/RedCards, Wins-Losses-Draws
Sean Johnson,    GK,  19 +0, 53, 33, -,      -,  -,  0-2,  0y/0r, 0-0-0
Greg Cochrane,   DF,  11 +0,  -,  -, 0,  4 (1),  2,  6-2,  2y/0r, -
Quincy Amarikwa, FW,  15 +2,  -,  -, 6, 37 (15), 3, 33-45, 4y/1r, -

  Let me know what you think or if I'm missing something. Cheers.
