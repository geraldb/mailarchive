Hello,

   Great stuff. No worries I try to rework the addon a little latter
today (e.g. for gems I use a rake task helper, that is, hoe [1] -
makes it easier to manage gems and I m old school e.g. requires using
a Manifest.txt to double-check the gem package etc.) and so on.


   For the player stats I suggest keeping the stats separate from you
squads, that is, store the stats in a  *new* (that is, different /
separate ) file (s). (e.g. maybe one new file per squad?  )


  Why?

  - Modular design, that is, lets you use only what you want or need
(e.g. not everyone needs/wants stats) or might use a different stats
format etc.

  - You can use / build a SquadStats reader/parser or PlayerStats
parser from scratch - no need to "monkey patch" it into the
SquadsReader - makes it easier to change / maintain etc.

  - You can use whatever format you think best (might even be just a
.csv - comma-separated values file) for easy import.

  Keep it up. Cheers.
