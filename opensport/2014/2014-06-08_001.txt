Hello,

 Just a shorter answer if the first was too confusing. Sorry.

>Any plans to allow the sports db to import other formats like json or csv?

  It's an open source project. You're more than welcome to add a json
or csv importer (in C# or whatever is your language of choice).  It's
a great addition and welcome. You can see the "raw" database table
schema [1] in the Ruby gem or, of course, download an SQLite copy [2]
and browse the tables and schema w/ SQLite Studio (free, open-source
single-exe tool) or similar to see it without the Ruby flavor.

 Cheers.

[1] github.com/sportdb/sport.db.ruby/blob/master/lib/sportdb/schema.rb
[2] github.com/openfootball/build/releases
[3] sqlitestudio.pl
