Hello,

   Welcome to football.db.

   The datasets are just text and, thus, on purpose (and by design)
language-independent so you can in theory use whatever programming
language you like.

   However, to read and import the datasets into an SQL database (of
your choice) you will need a parser / tool.

    As there's only so much I can do the official command line tools
(called sportdb [1]) are coded in ruby  There might be some python
tools in various stages somewhere out on the internet or you might ask
around.

     I'd say the easiest might be to build yourself an SQL database
with the sportdb command line tool (in ruby) using a datafile [2] and
than use python for your app or data analysis or whatever you plan to
code.

   Cheers. Prost.

[1] https://github.com/sportdb/sport.db
[2] https://github.com/openfootball/datafile
