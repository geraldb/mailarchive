Hello Ben,

   Welcome to football.db. Thanks for your kind words.

   Sorry for the missing and confusing (sometimes outdated) docu -
there's onyl so much I can do. Anyways,

    To install the command line tool [1] - all the machinery is in
Ruby. Packages in Ruby are called gem (RubyGems) it's a wordplay on
Ruby.

    Anyways, once you have ruby installed (it's shipping with Appe
MacOS, by the way built-in).

   You can install the machinery with:

$ gem install sportdb

   Once installed try:

$ sportdb help

   To see if you now have the machinery in place.

To setup and parse the world cup datesets use a datafile [2] (it's
built into the sportdb command line tool), thus, try:

$ sportdb new worldcup

   That should download the datasets (in a zip), setup an SQLite
sport.db database and parse all datasets (from the local zip).

   That's it.

   Let us know who it goes or if that makes any sense. Cheers.

[1] https://rubygems.org/gems/sportdb
[2] https://github.com/openfootball/datafile/blob/master/worldcup.rb
